## 第一部分：Python 基础回顾

### 1.1 变量和数据类型

Python中有几种基本的数据类型：

```python
# 字符串 - 文字信息
name = "北京"

# 整数 - 数字（没有小数点）
temperature = 25

# 浮点数 - 数字（有小数点）
humidity = 65.5

# 布尔值 - 真或假
is_raining = True

# 列表 - 一组数据
cities = ["北京", "上海", "广州", "深圳"]

# 字典 - 键值对
weather_info = {
    "city": "北京",
    "temperature": 25,
    "condition": "晴天"
}
```

**比喻**：变量就像是一个个盒子，每个盒子有一个名字（变量名），里面可以放不同类型的东西。

### 1.2 函数 - 加工机器

函数就像一个有特定功能的小机器：
- 你给它输入（参数）
- 它帮你完成工作
- 它给你输出（返回值）

**语法**：
```python
def 函数名(参数1, 参数2):
    """这是说明文档"""
    # 做一些事情
    return 结果
```

**示例**：
```python
def 打招呼(名字):
    """向指定的人打招呼"""
    return f"你好，{名字}！"

# 使用函数
结果 = 打招呼("小明")
print(结果)  # 输出：你好，小明！
```

**图示**：
```
输入(参数) → [函数: 加工] → 输出(返回值)
  "小明"   →   打招呼    →  "你好，小明！"
```

### 1.3 类 - 蓝图和对象

类就像一个"蓝图"或"模板"：
- 它定义了某种东西应该有什么**属性**（数据）
- 它定义了这种东西能做什么**方法**（功能）

**比喻**：
- **类** = 汽车设计图纸（蓝图）
- **实例** = 根据图纸造出来的具体汽车

**示例**：
```python
class 机器人:
    """这是一个机器人的类"""
    
    # __init__ 是初始化方法，创建对象时会自动调用
    def __init__(self, 名字):
        self.名字 = 名字  # 属性
        self.状态 = "待机"  # 属性
    
    # 方法：机器人能做什么
    def 说话(self, 内容):
        """让机器人说话"""
        print(f"{self.名字}: {内容}")
    
    def 移动(self, 方向):
        """让机器人移动"""
        print(f"{self.名字}向{方向}移动")
        self.状态 = "移动中"

# 创建机器人对象
小智 = 机器人("小智")

# 调用方法
小智.说话("你好！")  # 输出：小智: 你好！
小智.移动("北")       # 输出：小智向北移动
```

**图示**：
```
类（蓝图）
┌─────────────────────────┐
│ 机器人                   │
├─────────────────────────┤
│ 属性：                   │
│   - 名字                 │
│   - 状态                 │
├─────────────────────────┤
│ 方法：                   │
│   - 说话()               │
│   - 移动()               │
└─────────────────────────┘
         ↓ 创建
实例（具体对象）
┌─────────────────────────┐
│ 小智（机器人）            │
├─────────────────────────┤
│ 属性值：                 │
│   - 名字 = "小智"         │
│   - 状态 = "待机"         │
├─────────────────────────┤
│ 可以执行：               │
│   - 小智.说话()          │
│   - 小智.移动()          │
└─────────────────────────┘
```

### 2.1 天气查询函数

让我们逐步拆解 `get_weather` 函数：

```python
def get_weather(city: str) -> str:
    """
    通过调用 wttr.in API 查询真实的天气信息。
    
    参数：
        city: 城市名称，比如 "北京"、"上海"
    返回：
        天气描述字符串，比如 "北京当前天气:晴天，气温25摄氏度"
    """
    # 第一步：构建请求URL
    url = f"https://wttr.in/{city}?format=j1"
    # 如果 city = "北京"，则 url = "https://wttr.in/北京?format=j1"
    
    try:
        # 第二步：发送网络请求
        response = requests.get(url)
        
        # 第三步：检查请求是否成功
        response.raise_for_status()
        # 如果不成功（比如404），会抛出异常
        
        # 第四步：解析返回的JSON数据
        data = response.json()
        # wttr.in 会返回类似这样的数据：
        # {
        #     "current_condition": [
        #         {
        #             "weatherDesc": [{"value": "晴天"}],
        #             "temp_C": "25"
        #         }
        #     ]
        # }
        
        # 第五步：提取需要的信息
        current_condition = data['current_condition'][0]
        weather_desc = current_condition['weatherDesc'][0]['value']
        temp_c = current_condition['temp_C']
        
        # 第六步：返回格式化后的字符串
        return f"{city}当前天气:{weather_desc}，气温{temp_c}摄氏度"
    
    except requests.exceptions.RequestException as e:
        # 网络相关的错误
        return f"错误:查询天气时遇到网络问题 - {e}"
    
    except (KeyError, IndexError) as e:
        # 数据解析相关的错误
        return f"错误:解析天气数据失败，可能是城市名称无效 - {e}"
```

**流程图**：
```
用户调用 get_weather("北京")
         ↓
    构建请求URL
         ↓
    发送网络请求
         ↓
    ┌─────┴─────┐
    ↓           ↓
  成功        失败
    ↓           ↓
  解析数据    返回错误信息
    ↓
  提取信息
    ↓
  格式化返回
    ↓
  "北京当前天气:晴天，气温25摄氏度"
```

### 2.2 System Prompt - 给AI的说明书

System Prompt 就像给 AI 助手的一份详细工作说明：

```python
AGENT_SYSTEM_PROMPT = """
你是一个智能旅行助手。你的任务是分析用户的请求，并使用可用工具一步步地解决问题。

# 可用工具:
- `get_weather(city: str)`: 查询指定城市的实时天气。
- `get_attraction(city: str, weather: str)`: 根据城市和天气搜索推荐的旅游景点。

# 输出格式要求:
你的每次回复必须严格遵循以下格式：

Thought: [你的思考过程和下一步计划]
Action: [你要执行的具体行动]

Action的格式必须是以下之一：
1. 调用工具：function_name(arg_name="arg_value")
2. 结束任务：Finish[最终答案]
"""
```

**作用说明**：

1. **定义角色**："你是一个智能旅行助手"
   - 让AI知道自己应该扮演什么身份

2. **列出工具**："你有这些能力可用"
   - 让AI知道它可以调用哪些函数

3. **规定格式**："必须这样输出"
   - 让AI的回答格式统一，方便程序解析

**比喻**：
```
System Prompt 就像是新员工入职手册：
┌─────────────────────────────┐
│ 新员工手册                   │
├─────────────────────────────┤
│ 职位：智能旅行助手            │
│                             │
│ 可用工具：                   │
│ ✓ 天气查询系统              │
│ ✓ 景点推荐搜索引擎          │
│                             │
│ 工作方式：                   │
│ 1. 先思考，再行动           │
│ 2. 每次只做一件事           │
│ 3. 完成后要汇报             │
└─────────────────────────────┘
```

### 2.3 智能体的思考-行动-观察循环

这是智能体最核心的机制，模拟人类解决问题的方式：

```
┌──────────────────────────────────────────────────────┐
│            智能体工作流程                              │
└──────────────────────────────────────────────────────┘

   用户请求: "北京天气怎么样？适合去哪里玩？"
              ↓
┌─────────────────────────────────────────┐
│  思考 (Thought)                          │
│  "用户想知道天气，我应该先查询天气"       │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  行动 (Action)                          │
│  get_weather(city="北京")               │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  观察 (Observation)                      │
│  "北京当前天气:晴天，气温25摄氏度"        │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  思考 (Thought)                          │
│  "已获取天气信息，现在应该搜索景点推荐"    │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  行动 (Action)                          │
│  get_attraction(city="北京", weather="晴天") │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  观察 (Observation)                      │
│  "推荐故宫、颐和园、天安门广场..."         │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  思考 (Thought)                          │
│  "信息收集完毕，可以给用户完整答案了"      │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  行动 (Action)                          │
│  Finish[北京今天晴天，适合去故宫...]      │
└─────────────────────────────────────────┘
              ↓
        最终答案输出给用户
```

**为什么需要循环？**
- 因为复杂任务不能一步完成
- 需要"边做边想"，根据结果调整策略
- 就像人做事一样：做一点，看结果，再决定下一步

### 2.4 正则表达式 - 文字搜索引擎

正则表达式是一种强大的"文字匹配"工具：

```python
import re

# 假设AI返回了这样的文字：
llm_output = """
Thought: 我需要先查询天气
Action: get_weather(city="北京")
"""

# 使用正则表达式提取信息
# r'模式' 中的 r 表示原始字符串

# 1. 提取 Thought 后面的内容
thought = re.search(r'Thought:\s*(.+?)(?=Action:|$)', llm_output)
if thought:
    print(thought.group(1))  # 输出：我需要先查询天气

# 2. 提取 Action 后面的内容
action = re.search(r'Action:\s*(.+)', llm_output)
if action:
    print(action.group(1))  # 输出：get_weather(city="北京")

# 3. 检查是否是 Finish
finish = re.search(r'Finish\[(.+)\]', action.group(1))
if finish:
    print("任务完成！答案：" + finish.group(1))
```

**常用模式解释**：
```
.        - 任意单个字符
.*       - 任意多个字符
\w       - 字母、数字、下划线
\d       - 数字
\s       - 空白字符
(.*?)    - 匹配内容并捕获（非贪婪）
(?=...)  - 前瞻断言
```

### 2.5 异常处理 - 保险机制

代码可能出错，异常处理就像"保险"：

```python
try:
    # 尝试执行这段代码
    response = requests.get(url)
    data = response.json()
    result = data['current_condition'][0]

except requests.exceptions.RequestException as e:
    # 如果发生网络错误，执行这里
    print(f"网络出错了: {e}")
    result = "无法获取数据"

except (KeyError, IndexError) as e:
    # 如果数据格式不对，执行这里
    print(f"数据解析出错: {e}")
    result = "数据格式错误"

finally:
    # 无论是否出错，都会执行这里
    print("无论成功失败，我都会执行")
```

**比喻**：
```
try:    "我先试试这样做"
except: "如果失败了，我来补救"
finally: "最后我一定要做这个收尾"
```

## 第五部分：常见问题

### Q1: 为什么要使用环境变量？

**答**：保护敏感信息（如API密钥）。如果直接写在代码里，会被其他人看到，存在安全隐患。

### Q2: 什么是 API？

**答**：Application Programming Interface（应用程序编程接口）。简单说就是"服务窗口"，你向它发送请求，它返回结果。

### Q3: 什么是 JSON？

**答**：JavaScript Object Notation。一种轻量级的数据格式，方便不同系统之间传递数据。

### Q4: 为什么需要循环限制？

**答**：防止智能体陷入无限循环。如果任务无法完成，设置了最大循环次数后程序会自动停止。
